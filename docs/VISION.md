# VISION.md — Inventra

## What We're Building

**Inventra** is a modern inventory and business management system designed for tech/gadget retail businesses. It combines powerful inventory tracking, point-of-sale functionality, and comprehensive business analytics in one sleek, animated interface.

### Two Deployments

| Version | Repo | Visibility | Purpose | Language |
|---------|------|------------|---------|----------|
| **Inventra** | `inventra` | Public | Portfolio showcase | English |
| **JDUA COMPUTER** | `jdua-computer` | Private | Real business use | Bahasa Indonesia |

**Portfolio Version** includes:
- Landing page (marketing/showcase)
- Simulated authentication
- Full app with demo data (localStorage)
- Hosted on Vercel

**JDUA COMPUTER Version** includes:

- Real local database (SQLite)

## Design Philosophy

### Aesthetic Direction: "Industrial Warmth"

A bold, playful, handcrafted design language that feels like it was built by creative developers who care — not generated by AI. Think modern factory floor meets startup energy: textured, tactile, warm, and alive. Every surface has character. Every animation has personality. The whole experience should make people think *"real humans with taste made this."*

We're inspired by hex.tech's ability to be simultaneously grungy and corporate, playful and professional, dynamic and functional. That duality is our north star.

### Visual DNA

| Attribute | Description |
|-----------|-------------|
| **Tone** | Confident, playful, opinionated — a startup that knows what it's doing and isn't afraid to show personality |
| **Visual Style** | Industrial-modern warmth meets geometric precision. Textured, tactile, alive — never flat, sterile, or generic |
| **Color Strategy** | Warm cream canvas + warm charcoal depth + dusty rose accents. Never pure white or pure black. Every color has warmth |
| **Texture Philosophy** | Backgrounds should feel *physical* — subtle grain, noise overlays, fine dot-grid patterns (like hex.tech). Every surface has character and depth |
| **Motion Philosophy** | Dynamic, playful, and handcrafted — marquee tickers, parallax layers, staggered reveals, interactive product demos. Animations feel human-made and surprising, never formulaic or algorithmic |
| **Target Audience** | Gen-Z entrepreneurs, tech-savvy retail business owners, startup-minded people who appreciate design with personality |

### Design Differentiators

1. **Scrolling Marquee Ticker** — Continuous horizontal scroll bands with features/announcements on a dark warm charcoal bar with dusty rose text, adds constant dynamic energy (à la hex.tech top bar)
2. **Warm Textured Surfaces** — Grain overlays, dot-grid patterns, warm-toned backgrounds that feel tactile and physical. Every surface has depth — never a plain solid color
3. **Interactive Product Demos** — Real product UI embedded as hero content. Tabbed views showing actual app screens with live data, not abstract illustrations or mockups
4. **Parallax Depth System** — Multiple scroll-speed layers creating 3D depth with GSAP ScrollTrigger
5. **Bold Type Mixing** — Confident pairing of italic serif display with bold geometric sans-serif in the same headline. Varying scale dramatically. Big statements that draw you in
6. **Handcrafted Motion** — Animations that feel human and surprising: staggered text reveals, playful hover states, organic easing curves, scroll-triggered choreography
7. **Industrial Chrome Details** — Subtle borders, fine divider lines, badge elements, pill shapes, and UI chrome that evoke workshop precision. Small details that show obsessive care

## Typography

**No AI slop fonts.** We use distinctive, quality typefaces with character. Bold type mixing is encouraged — serifs and sans-serifs living together confidently, just like hex.tech's hero section.

### Recommended Pairings

| Use Case | Primary Options | Fallback |
|----------|-----------------|----------|
| **Display/Headlines** | Fraunces (serif, characterful — use italic for hero), Space Grotesk (geometric, bold), Sora (clean, modern) | system-ui |
| **Body Text** | Satoshi (warm, readable), Inter Tight (compact, clean), Plus Jakarta Sans (friendly, professional) | system-ui |
| **Monospace/Data** | JetBrains Mono, IBM Plex Mono | monospace |
| **Accent/Labels** | Space Grotesk (uppercase, tight tracking), Satoshi (medium weight, small caps) | system-ui |

### Type Mixing Strategy

Inspired by hex.tech's confident typography approach:
- **Hero Headlines**: Mix italic serif (Fraunces Italic) with bold sans-serif (Space Grotesk) in the same heading — like hex.tech's "Bring the magic of AI / to data, for everyone" where the first line is italic serif and the second is bold sans-serif
- **Section Headlines**: Go BIG. Use dramatic scale. One family per headline is fine, but vary between sections
- **Subheads**: Use the dusty rose accent color for subtitle text, contrasting with the dark headline
- **Body**: Always clean and readable. Satoshi or Plus Jakarta Sans at comfortable sizes
- **Labels/Badges**: Uppercase, wide letter-spacing, smaller weight for industrial feel (like hex.tech's "BLOG" label in dusty rose)
- **Data/Tables**: JetBrains Mono for numbers, monospace alignment in data-heavy views

### Selection Criteria
- Must be Google Fonts or self-hostable (free)
- Must have sufficient weight variations (400, 500, 600, 700 minimum)
- Must render well at small sizes for data tables
- Must feel modern, not dated
- Must have *character* — if it could be any font, it's the wrong font

## Color System

### Philosophy

Inspired directly by hex.tech's color approach: **warm cream canvas + warm charcoal depth + dusty rose accent**. No cold blues, no pure whites, no pure blacks. Every color has warmth and personality.

The palette revolves around three pillars:
1. **Warm Cream** — the canvas. Light backgrounds that feel like textured paper, not a blank screen
2. **Warm Charcoal** — the depth. Dark tones with warm rose/brown undertones, never cold or blue-shifted
3. **Dusty Rose** — the accent. A sophisticated, muted blush used sparingly for highlights, labels, interactive elements, and personality

### Surface Treatments

The core of our visual identity. Every surface should feel *physical*, not flat.

```css
/* Grain/Noise Overlay — apply to backgrounds for texture */
--grain-opacity: 0.03;           /* Light mode: very subtle */
--grain-opacity-dark: 0.05;      /* Dark mode: slightly more visible */

/* Dot-Grid Pattern — like hex.tech's hero background */
/* Implement as a subtle repeating SVG dot grid at very low opacity */
--dot-grid-size: 24px;           /* Spacing between dots */
--dot-grid-color: rgba(28, 25, 23, 0.06);        /* Light mode dots */
--dot-grid-color-dark: rgba(245, 240, 235, 0.04); /* Dark mode dots */

/* Warm Glass — used sparingly for floating elements */
--glass-bg: rgba(250, 248, 245, 0.7);           /* Warm cream glass */
--glass-bg-dark: rgba(34, 30, 32, 0.7);         /* Dark warm glass */
--glass-border: rgba(224, 218, 211, 0.5);        /* Warm border */
--glass-border-dark: rgba(53, 47, 49, 0.5);      /* Dark warm border */
--glass-blur: blur(12px);

/* Industrial Dividers */
--divider: 1px solid var(--border);
--divider-strong: 2px solid var(--border);
```

### Color Usage Guidelines

| Element | Light Mode | Dark Mode |
|---------|-----------|-----------|
| **Page background** | `--background` (#F5F1EC) + grain/dot-grid | `--background-dark` (#141113) + grain |
| **Alternating sections** | Swap `--background` ↔ `--background-alt` | Swap `--background-dark` ↔ `--background-alt-dark` |
| **Cards** | `--surface` with `--border`, soft shadow | `--surface-dark` with `--border-dark` |
| **Modals/Dropdowns** | `--surface-raised` with stronger shadow | `--surface-raised-dark` |
| **Navbar/Sticky headers** | Warm glass + blur | Dark warm glass + blur |
| **Marquee ticker bar** | `--ticker-bg` with `--ticker-text` | Same (always dark) |
| **Primary CTA buttons** | `--accent-500` bg, cream/white text | `--accent-500` bg, cream/white text |
| **Secondary CTA buttons** | `--surface` bg with `--accent-500` border | `--surface-dark` bg with `--accent-500` border |
| **Ghost/Subtle buttons** | Transparent with warm border, accent hover | Transparent with warm border, accent hover |
| **Links** | `--accent-600` (darker for contrast on cream) | `--accent-400` (lighter for contrast on dark) |
| **Labels/Tags** | `--accent-500` text, uppercase, tracked | `--accent-400` text, uppercase, tracked |
| **Active/Selected states** | `--accent-50` bg tint + `--accent-500` indicator | `--accent-900` bg tint + `--accent-400` indicator |
| **Headlines** | `--text` warm near-black | `--text-dark` warm cream |
| **Subheadlines** | `--accent-500` or `--text-muted` | `--accent-400` or `--text-muted-dark` |

### Dark Section Bands

Like hex.tech, we can use dark-mode colors inside light-mode pages for dramatic section contrast:
- Hero sections, CTA bands, testimonial sections can use `--background-dark` even in light mode
- Text becomes `--text-dark` (warm cream)
- Accents become `--accent-400` (lighter rose for dark backgrounds)
- This creates visual rhythm and drama as you scroll

## Motion Principles

### Animation Hierarchy

| Priority | Type | Duration | Easing |
|----------|------|----------|--------|
| 1 | Page transitions | 0.5-0.8s | `cubic-bezier(0.4, 0, 0.2, 1)` |
| 2 | Section reveals | 0.6-1.0s | `cubic-bezier(0.16, 1, 0.3, 1)` |
| 3 | Component enters | 0.3-0.5s | `ease-out` |
| 4 | Micro-interactions | 0.15-0.25s | `ease-in-out` |
| 5 | Marquee/Ticker | Continuous | `linear` (constant speed) |

### Signature Animations

| Animation | Where | Description |
|-----------|-------|-------------|
| **Marquee Ticker** | Top bar, section dividers, CTA bands | Continuous horizontal scroll of text/badges on dark `--ticker-bg` with `--ticker-text` dusty rose. Pauses on hover |
| **Staggered Text Reveals** | Headlines, hero section | Words or letters animate in sequentially. GSAP SplitText style |
| **Parallax Layers** | Hero, between sections | Elements at 0.5x, 1x, 1.5x scroll speeds via GSAP ScrollTrigger |
| **Scroll-Triggered Entrances** | All sections | Elements fade/slide in as they enter viewport. Stagger children for wave effect |
| **Interactive Demos** | Dashboard preview, feature showcases | Tab switching with animated content transitions |
| **Counter Animations** | Stats, KPIs | Numbers count up when visible |

### Scroll Behavior
- **Native scrolling** preserved for best touchpad/trackpad experience
- CSS `scroll-behavior: smooth` for anchor links
- Parallax layers at 0.5x, 1x, 1.5x speeds via GSAP ScrollTrigger
- Scroll-triggered reveals via GSAP ScrollTrigger
- Respects `prefers-reduced-motion` user preference

### Interactive States
- Hover: Scale (1.02-1.04), warm glow or subtle lift, accent tint background
- Active: Scale (0.98), brief satisfying press effect
- Focus: Ring with `--accent-500` dusty rose, accessible outline
- Card hover: Lift with shadow increase, subtle border color warm-shift
- Button hover: Background shift + slight scale, playful but controlled

## Accessibility Commitments

- WCAG 2.1 AA compliance minimum
- Keyboard navigation for all interactive elements
- Screen reader friendly (proper ARIA labels)
- Reduced motion support (`prefers-reduced-motion`)
- Sufficient color contrast ratios (dusty rose accent passes AA on both cream and charcoal)
- Focus indicators always visible (dusty rose ring)

## Quality Standards

- TypeScript strict mode
- ESLint + Prettier enforced
- Component-based architecture

### UI Quality
- Responsive (mobile, tablet, desktop)

- Loading states (skeletons, spinners)
- Empty states with helpful messaging
- Error states with recovery actions
- Toast notifications for feedback
- **Every background has texture** — no flat solid colors

### Performance
- Lighthouse score > 90
- Lazy loading for images
- Code splitting per route
- Optimized animations (GPU-accelerated)
- No layout shifts (CLS near 0)
- Grain/noise/dot-grid textures are lightweight (SVG or CSS, not heavy PNGs)

## The Golden Rule

> **Always use the `frontend-design` skill before any UI work.**
>
> Read `/mnt/skills/public/frontend-design/SKILL.md` before creating any component, page, or visual element. This ensures we maintain creative excellence and avoid AI slop.

Every design decision must be:
- **Intentional** — serves a purpose
- **Distinctive** — memorable and unique
- **Textured** — has physical depth and warmth
- **Warm** — cream, charcoal, and dusty rose — never cold or sterile
- **Handcrafted** — feels human-made, not generated
- **Cohesive** — fits the overall system
- **Polished** — refined to the smallest detail

> If it looks like it could have been made by any AI tool, it's not good enough. If it looks like hex.tech's team could have built it, we're on the right track.
